Бот для поиска изображений в чатах по их текстовому описанию. Бот не видит сообщения до добавления в чат, ведётся разработка функции импорта. Я не храню на своей стороне обрабатываемые изображения, только их закодированное представление и уникальный идентификатор. Бот не работает в каналах, но может работать в комментариях.

/start, ping - Тест ответа бота, его статуса онлайн и корректности настройки в чате

/help - Это справочное сообщение

/find или /найди [топ 5] &lt;описание&gt; [от username] [с YYYY.MM.DD] [по YYYY.MM.DD] [с текстом "..."] - Поиск изображений по текстовому описанию на русском или английском языке, описание фильтров:
<code>[топ 5]</code> - число изображений в выдаче, не более 100 (опционально, по умолчанию 5).
<code>[от username]</code> - фильтр по пользователю (опционально).
<code>[с YYYY.MM.DD] [по YYYY.MM.DD]</code> - фильтр по дате (опционально оба).
<code>[с текстом "..."]</code> - фильтр по распознанному тексту (опционально).
<b>Совет:</b> делайте описание так, как если бы это был промпт для генерации изображения в Stable Diffusion.

/import + zip-архив - Индексация истории чата до добавления бота с помощью zip-архива с историей чата в JSON-формате и изображениями. Ограничение: не более 20 Мб на файл.

/setcontext [заголовок чата] - Изменить контекст поиска в личных сообщениях с ботом на поиск по чату. Полезно, чтобы не засорять основной чат. Вызов без аргументов сбрасывает контекст чата на личный
__
Используемые технологии:
- <a href="https://huggingface.co/openai/clip-vit-large-patch14">CLIP</a> для обработки изображений
- <a href="https://huggingface.co/facebook/wmt19-ru-en">FairSeqMachineTranslation</a> для перевода запроса с русского на английский
